## 1. 结论速览（本仓库锁定版本）

| 组件 | 锁定版本 | 说明 |
|---|---|---|
| JDK | **21 (LTS)** | 用户环境统一要求；SAA 官方最低要求 JDK 17+ |
| Maven | 3.9+ | 多模块工程，禁用 Gradle |

| Spring AI Alibaba (SAA) | **1.1.2.2** | 官方最新稳定版（1.1.2.1 因缺陷被官方撤回推荐） |
| Spring AI Alibaba Extensions | **1.1.2.2** | 随主线统一版本，DashScope 模型、MCP Registry 等扩展实现 |

| ~~1.1.2.1~~ | 2026-03-09 | 1.1.2 | 3.5.x | **官方标记为缺陷版本**，Release 页明示"请改用 1.1.2.2" |

> **版本更正说明**：Phase 1 首次调研（依据 java2ai.com 版本说明页与搜索快照）曾将当前推荐版记录为 1.1.2.0。经对 GitHub Releases 的直接核验，官方最新稳定版实为 **1.1.2.2**（1.1.2.1 因缺陷被官方撤回推荐）。本报告与父 POM 已同步修正，1.1.2.0 与 1.1.2.2 的 API 差异仅为新增（AgentScope 集成、多智能体模式官方样例），不影响 1.1.2.0 时期已写章节的代码兼容性。

### 2.1.1 1.1.2.2 相对 1.1.2.0 的增量（不构成破坏性变更）

- **AgentScope Java 集成**：新增 `spring-ai-alibaba-starter-agentscope`，提供 `AgentScopeAgent`，可将 [AgentScope](https://github.com/agentscope-ai/agentscope-java)（另一个 Java Agent 框架）的 ReActAgent 包装为 SAA Graph 的 `BaseAgent` 节点混合编排；
- **官方多智能体模式样例集**（`examples/multiagent-patterns/`）：Subagent（主控委派专职子智能体）、Supervisor（中枢智能体将 Calendar/Email 智能体封装为 AgentTool 调用）、Skills（`read_skill` 按需加载技能）、Routing（simple / graph 两种形态）、Handoffs（单智能体状态流转 / 多智能体图节点交接）、Workflow（RAG 与 SQL Agent 的图形化工作流范式）——本教程第 13/15 章将直接复用这些官方范式命名，避免自造术语；
- **多模态与语音 Agent**：Voice Agent（DashScope ASR + ReactAgent + CosyVoice TTS 的三明治架构，WebSocket 流式）、Multimodal（视觉理解 + Wanx 图像生成 + TTS 工具化）。这两项本教程列为可选扩展阅读，不占用第 13~15 章正课篇幅（不在用户列出的必修范围内）。

依赖坐标示例（`spring-ai-alibaba-bom` 与 `spring-ai-alibaba-extensions-bom` 需**同时导入**——社区曾反馈 1.1.2.0 主 BOM 未收录 `spring-ai-alibaba-starter-dashscope` 的版本管理（见 [alibaba/spring-ai-alibaba#4030](https://github.com/alibaba/spring-ai-alibaba/discussions/4030)），同时导入两个 BOM 是官方文档给出的稳妥写法，本仓库父 POM 已按此配置）：

```xml
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-agentscope</artifactId>
    <!-- 版本由 spring-ai-alibaba-bom 统一管理 -->
</dependency>
```

### 2.2 1.1.x 相对 1.0.x 的结构性变化（重点补齐项）

这是从"早期 Demo 经验"升级到当前体系时**必须重建心智模型**的部分：

1. **模块体系重组**：核心能力拆分为
   - `spring-ai-alibaba-agent-framework` —— 高层智能体框架（ReactAgent、多智能体编排、Hooks、Skills、Human-in-the-Loop、Context Engineering）；
   - `spring-ai-alibaba-graph-core` —— 底层图运行时（持久化、流式、编排），Agent Framework 构建于其上，定位类似 LangGraph 之于 LangChain；
   - DashScope 等模型接入迁移至独立仓库 **spring-ai-extensions**；其版本由 `spring-ai-alibaba-extensions-bom` 管理，本仓库父 POM 与主 BOM 一并导入，避免个别模块版本缺失的问题（见 §2.1.1）。
2. **官方主推开发路径变化**：从"直接写 Graph"转向"优先使用 Agent Framework 内置模式（Sequential / Parallel / Routing / Loop / Supervisor / Handoffs），复杂编排才下沉到 Graph API"。
3. **平台化组件成型**：`spring-ai-alibaba-admin`（一站式 Agent 平台：可视化开发、可观测、评测、MCP 管理，支持 Dify DSL 迁移）、`spring-ai-alibaba-studio`（嵌入式调试 UI）、`spring-ai-alibaba-sandbox`（Agent 沙箱运行时）。
4. **企业级云原生 Starter**：`spring-ai-alibaba-starter-a2a-nacos`（A2A 智能体互通）、`spring-ai-alibaba-starter-config-nacos`（动态配置 / 模型与 Prompt 热更新）、`spring-ai-alibaba-starter-graph-observation`（Micrometer / OpenTelemetry 可观测）、`spring-ai-alibaba-starter-builtin-nodes`（预置图节点）。
5. **生态项目更名与演进**：JManus → **Lynxe**；NL2SQL → **DataAgent**；DeepResearch 提供 Graph 版独立发行版；新增 **AssistantAgent** 企业助手框架。

### 2.3 1.1.2.0 版本新增能力（2026-02 发布，本教程默认写法基线）

| 方向 | 新能力 |
|---|---|
| Agent | ReactAgent 集成 **Agent Skills**（技能渐进式披露：系统提示只注入技能清单，模型按需 `read_skill` 加载完整 SKILL.md，降低 token 消耗）；Flow Agent 统一 Hooks；`streamMessages` API；工具 `returnDirect`；`ToolContextHelper` |
| Multi-agent | Supervisor / LlmRouting 支持**并行子智能体执行**；内置 Routing、Supervisor、Handoffs、Custom Workflow 模式最佳实践 |
| Graph | **并行条件边**；并行分支聚合策略（AllOf / AnyOf）；批量 addEdge；interruptAfter Hook；AgentToolNode 异步工具执行；流式节点完整输出 |
| 质量修复 | 序列化、MergeStrategy、systemPrompt、selectorModel、DashScopeApi 默认 HTTP Client（修复 Connection Reset）等 |
| Admin/UI | Flow UI 依赖由 `@agentscope-ai/flow` 更名为 `@spark-ai/flow`（升级注意项） |

## 3. Spring Boot 版本决策（重要风险披露）

用户约束为 **Spring Boot 3.x 最新稳定版**，且 SAA 1.1.2.x 官方对齐 Boot 3.5.x，因此锁定 **3.5.16**。但必须如实说明现状：

- 3.5.16 是 3.x 线**最后一个 OSS 版本**（2026-06-25 发布），3.5 线开源支持已于 **2026-06-30 到期**；
- Spring Boot 当前受支持线为 4.0.x / 4.1.x（基于 Spring Framework 7、Jackson 3）；
- **重要更正（2026-07-03 二次核验）**：Spring AI **2.0.0 已于 2026-06-12 正式 GA**（面向 Spring Boot 4.0/4.1 + Spring Framework 7，非此前记录的"仍处 Milestone"状态，此前表述已过期，特此更正）；
- 但 **SAA（Spring AI Alibaba）截至本次核验，最新版本仍是 1.1.2.2（2026-03-10），未发布任何对齐 Spring AI 2.0 / Spring Boot 4 的版本**——SAA 官方仓库自 3 月以来未见新 Release，其 1.1.2.2 官方对齐的仍是 Spring AI 1.1.2。这意味着：**选择 SAA 就意味着当前必须停留在 Boot 3.5.x 线**，这不是"我们主动选择保守"，而是"生态位上游（SAA）尚未跟进"这一客观限制——详见第 22 章《Spring AI 2.0 现状与迁移前瞻》的专门分析。

**决策**：教学与落地以"官方兼容矩阵内的组合"为最高优先级 → `Boot 3.5.16 + Spring AI 1.1.x + SAA 1.1.2.2`。这是当前**唯一同时满足"全部 GA、官方互相对齐"的组合**。同时：

- 本教程第 20+ 章将附《面向 Spring AI 2.0 / Boot 4 的前瞻与迁移准备》，梳理已公布的破坏性变更清单（Jackson 3、MCP annotation 包迁移、ToolContext 移除会话历史、options 类 setter 移除转 Builder 等），使工程具备平滑升级路径；
- 生产环境若对 CVE 响应有强合规要求，可评估 Tanzu 商业支持或提前规划 Boot 4 迁移窗口。

## 4. Spring AI 版本现状

### 4.1 三条发布流（截至 2026-07）

| 发布流 | 最新版本 | 状态 | 说明 |
|---|---|---|---|
| 1.0.x | 1.0.9 | GA 维护 | 首个 GA（2025-05），仅安全与缺陷修复 |
| **1.1.x** | **1.1.8**（2026-06-12） | GA 维护 | 本仓库所在线；SAA 1.1.2.2 对齐其中的 1.1.2 |
| 2.0.x | **2.0.0 GA（2026-06-12）** | 正式发布 | 面向 Boot 4.0/4.1 + Spring Framework 7 / Jackson 3，含大量破坏性变更；**SAA 尚无对齐版本，本仓库暂不可用，见第 22 章** |

### 4.2 1.1.2 与 1.1.8 的取舍

- SAA 官方 Release Notes 明确"本版本使用 Spring AI 1.1.2，请对齐"，故父 POM 默认 `spring-ai.version=1.1.2`；
- 1.1.3~1.1.8 为同一 minor 线的补丁版，累计包含 **CVE-2026-22729/22730、CVE-2026-41705/41712/41713、CVE-2026-41863** 等安全修复，语义化版本承诺向后兼容；
- **生产建议**：将 `spring-ai.version` 升至 1.1.8 并回归测试（本仓库 CI 阶段会提供两版本矩阵验证）。教程正文出现差异时会以「版本差异」标注。

### 4.3 1.x 线内需要注意的弃用信号

- `PromptChatMemoryAdvisor` 已在 1.x 维护版本中标记弃用，官方要求迁移到**显式 conversationId 的新 Memory Advisor API** —— 本教程 Memory / Advisor 章节直接采用新写法；
- 1.1 线开始全面推行 **Builder / 构造器不可变 Options**（各模型 Options 的 mutable setter 在 2.0 中移除），本教程所有 Options 一律使用 Builder 写法，天然兼容未来 2.0。

## 5. 模型服务调研（DashScope + DeepSeek）

| 接入方式 | 依赖坐标 | 用途 |
|---|---|---|
| DashScope（百炼） | `com.alibaba.cloud.ai:spring-ai-alibaba-starter-dashscope`（版本随 `spring-ai-alibaba-extensions-bom` 1.1.2.2 管理） | 主力：qwen-max / qwen-plus / qwen-turbo 对话、text-embedding-v4 向量、多模态；亦可直接调用百炼托管的 deepseek 系列 |
| DeepSeek 官方 API | `org.springframework.ai:spring-ai-starter-model-deepseek` | 直连 DeepSeek 开放平台（OpenAI 兼容协议），用于多模型切换 / 降级 / 成本路由实战 |
| OpenAI 兼容通道 | `spring-ai-starter-model-openai` + 自定义 base-url | 通用兜底方案，演示"任意 OpenAI 兼容网关"接入模式 |

环境变量约定（全仓库统一，禁止硬编码）：`AI_DASHSCOPE_API_KEY`、`DEEPSEEK_API_KEY`。

> 具体模型型号（如 qwen 与 deepseek 的当前旗舰版本、上下文窗口、价格）迭代极快，将在 Phase 2 的 ChatClient 章节编写时按当日官方价格页再核对一次，并在文档中标注核对日期，避免教程给出过期定价。
