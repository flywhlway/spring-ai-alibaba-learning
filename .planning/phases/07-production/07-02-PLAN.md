新建 `.github/workflows/ci.yml`（per D-01；仓库当前无 `.github/`）。

触发（D-02）：`push` 与 `pull_request` 指向默认分支（写 `main`；若仓库默认分支实际为 `master`，以 `git remote show` / 仓库设置为准并与文档一致）；另加 `workflow_dispatch`。

Jobs（均可 `runs-on: ubuntu-latest`；JDK 经 `actions/setup-java@v4`，`distribution: temurin`，`java-version: '21'`，`cache: maven`，并设置 `cache-dependency-path` 覆盖根/`common`/`starter`/`examples/*/pom.xml`/`projects/*/pom.xml`）：

1. **common-starter**：`mvn -B -pl common,starter -am clean install`（D-04）。
2. **examples-compile**：matrix 覆盖全部 `examples/[0-9]*-*/` 目录（discretion：动态生成 matrix JSON 或静态 48 项与 `examples/README.md` 对齐；优先动态 `ls` 少漂移）。每项：`mvn -B -f examples/${{ matrix.demo }}/pom.xml -DskipTests compile`。`strategy.fail-fast: false`，`max-parallel: 4`~`8`（Pitfall 4）。可选 `needs: common-starter` 以便先装公共模块到 runner cache（若并行冷启动过慢，改为 needs + 仍靠 maven cache）。
3. **projects-test**：对 `knowledge-qa-platform`、`office-agent-assistant`、`smart-cs-platform` 分别 `mvn -B -f projects/&lt;name&gt;/pom.xml test`（D-06）。不设 `AI_DASHSCOPE_API_KEY`。接受 GitHub-hosted 有 Docker 时 Testcontainers IT 真实执行（Pitfall 7）；若红则修 IT，禁止关闭 Docker 或删条件。
4. **quality-gate**：`bash scripts/quality-gate.sh`（D-09 blocking）。可 `needs: [common-starter]` 或独立（脚本自身会再 install）。失败必须导致 workflow fail。

禁止：默认 job 引用 `secrets.AI_DASHSCOPE_API_KEY`；在 ci.yml 跑 `uat-phase3.sh` 或全量 `spring-boot:run`（D-05）；`mvn -pl` 选择 examples/projects 模块名。

Pin 仅官方 `actions/checkout@v4` 与 `actions/setup-java@v4`（Package Legitimacy：Approved）。
