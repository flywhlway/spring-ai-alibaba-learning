  <behavior>
    - AccessDeniedExceptionHandler.handle(AccessDeniedException) 返回 Result.fail(CommonResultCode.FORBIDDEN)，HTTP 状态 403
    - AuthorizationDeniedException（@PreAuthorize 拒绝，extends AccessDeniedException）命中同一 handler，不进入 GlobalExceptionHandler.handleUnexpected
    - 无 spring-security 的 Demo（仅 @Import GlobalExceptionHandler）编译/启动不受影响
  </behavior>
  <action>
    根因（admin-rbac-403-vs-500.md）：方法级 `@PreAuthorize` 在 DispatcherServlet/AOP 内抛 `AuthorizationDeniedException`，不经 ExceptionTranslationFilter；`GlobalExceptionHandler` catch-all `Exception` → HTTP 500。RBAC 拒绝正确，缺 403 映射。

    **选定方案（禁止把 Security 类型硬塞进 GlobalExceptionHandler 方法签名）：**
    1. `common/pom.xml`：新增 `spring-boot-starter-security` **optional=true**（与现有 optional web/validation 同模式）。包合法性：Spring Boot BOM 管理的官方 starter，非第三方未知包。
    2. 新建 `AccessDeniedExceptionHandler`（`@RestControllerAdvice`，`@author flywhl`）——**显式对齐 PATTERNS analog `GlobalExceptionHandler`**：
       - 同构：`@ExceptionHandler` + `@ResponseStatus` + `Result.fail(CommonResultCode.*)` + warn/error 日志、**不**把 `ex.getMessage()`/堆栈回传客户端（对标 `handleUnexpected` / `handleMethodArgumentNotValid` 的响应契约）
       - 本 handler：`@ExceptionHandler(AccessDeniedException.class)` + `@ResponseStatus(HttpStatus.FORBIDDEN)` + `Result.fail(CommonResultCode.FORBIDDEN)`（枚举 1002，固定文案，勿附加 exception message）
       - `AuthorizationDeniedException` 是子类，**不要**再写第二个 handler（避免重复映射）
    3. `GlobalExceptionHandler`：仅更新类注释，声明 AccessDenied 由 sibling advice 处理；**不要**在 catch-all 里用反射/类名字符串 hack；**不要**给 common 强制非 optional 的 security 依赖。
    4. `SaaLearningAutoConfiguration`：增加嵌套 `@Configuration`，使用**字符串形式** `@ConditionalOnClass(name = "org.springframework.security.access.AccessDeniedException")` + `@Import(AccessDeniedExceptionHandler.class)`。**禁止** `@ConditionalOnClass(AccessDeniedException.class)`：starter 无 spring-security 依赖，optional 不传递，类字面量会导致 starter 编译失败。勿给 starter 加 security 依赖；勿在顶层无条件 `@Import`。
    5. TDD：先写 `AccessDeniedExceptionHandlerTest`（直接调 handler 方法断言 code=1002 / 或 MockMvc 若更顺手），RED→GREEN。
    6. 在 `06-VALIDATION.md` 登记门禁：G-06-10-handler、G-06-10-optional、G-06-10-starter、G-06-10-test、G-06-10-uat。

    **明确不做：** 不改 `HumanHandoffController` / InterruptionMetadata / resume 顺序；不改 DashboardAdminController 的 `@PreAuthorize`；不新建产品 gap 给 D-14/CR-01。
  </action>
